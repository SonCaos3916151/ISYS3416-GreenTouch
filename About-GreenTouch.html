Non-functional Requirements

Non-functional requirements are what the users and developers expect the system should do. These expectations include:

User Interfaces

  • User Interface in accordance to Nielsen's heuristics
  • Comprehensive yet easy to use

Performance standards

  • Availability
  • Response time
  • Security
  • Mobility
  • Maintainability

Constraints

  • Regulatory policies
  • Hardware Limitations
  • API
  • Reliability requirements
  • Help farmer monitor crops on a large scale.  
  • Map landscapes
  • Help farmers with weather forecasts to help prevent or minimize losses caused by natural disaster or harsh weather.
  • Help farmers connect virtually with agricultural communities
  • Improve and innovate farming practices.
  • Does not fully automate farming processes  
  • Cannot contact appropriate authorities in case of emergencies happening to the farmlands (fires, accidents, etc.)
  • Cannot help farmers harvest expediently on a large scale
  • Cannot  mitigate damage once disasters have struck.
